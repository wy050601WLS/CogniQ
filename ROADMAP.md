# 知识问答系统 - 功能扩展规划

## 项目现状

系统已完成从"知识库模式"到"直接文件模式"的架构改造，当前功能状态：

### 已完成
- **文件管理**：上传/编辑/删除/替换文件，多格式支持（PDF/Word/MD/TXT/HTML）
- **知识广场**：浏览公开文件，添加引用模式（非复制，节省存储）
- **智能问答**：自动匹配最相关文件，流式响应，来源引用
- **多轮对话**：上下文记忆，对话历史管理
- **筛选排序**：文件格式、上传人、标签多维度筛选 + 热度/时间/大小排序
- **深色模式**：全站深色模式支持
- **认证安全**：JWT 双 Token 认证，自动刷新，登录回跳
- **文件引用**：user_files 关联表，引用文件可查看/删除但不可修改
- **标签系统**：文件标签管理，按标签筛选
- **版本控制**：文件版本历史，支持回滚
- **管理后台**：用户管理、文件管理、数据统计、系统设置
- **帮助中心**：分类帮助文档，搜索和 Markdown 渲染

## 一、短期扩展（1-2周）

### 1.1 文件描述 AI 自动生成

**当前问题：** 文件描述需手动填写，用户可能忽略。

**实现方案：**
- 上传文件时提供"AI 生成描述"选项
- 异步调用 LLM 提取文件关键信息
- 生成后缓存，不重复调用
- 前端显示生成进度

**预期效果：** 自动为文件生成准确描述，提升文件检索效率。

---

### 1.2 对话导出增强

**当前问题：** 对话导出格式较简单。

**实现方案：**
- 支持 PDF 导出（含格式和来源引用）
- 支持 Markdown 导出（完整格式）
- 前端增加导出格式选择
- 导出文件包含来源链接

**预期效果：** 用户可将有价值对话保存为高质量文档。

---

### 1.3 引用追溯优化

**当前问题：** 来源引用只显示片段，无法跳转。

**实现方案：**
- 在来源数据中存储文件 ID 和 chunk 位置
- 点击来源跳转到文件详情页
- 高亮显示引用的原始段落

**预期效果：** 用户可直接查看引用来源的完整上下文。

---

## 二、中期扩展（1-2月）

### 2.1 文件描述 AI 优化

- 基于文件内容自动生成更精准的描述

- 根据文件内容自动推荐标签
- 基于用户行为学习标签偏好
- 批量标签管理

### 2.4 多轮对话记忆优化

- 自动识别对话主题切换
- 智能管理对话上下文长度
- 对话摘要功能
